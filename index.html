Hospital API

Types of Users

  • Doctors
  • Patients

Doctors

Upon each patient visit, Doctors can authenticate their access and proceed with the following 2-step protocol:

  1. Register the patient
  2. Create a Report

Patient Report Fields

  • Created by doctor
  • Status Can be either of: [Negative, Travelled-Quarantine, Symptoms-Quarantine, Positive-Admit]
  • Date

Available Routes

Route Description
/doctors/register To register as a doctor, a username and password should be sent through a POST request to /doctors/register which returns a success message response.
/doctors/login To log in, a POST request should be sent to /doctors/login, which returns a JWT token upon successful authentication.
/patients/register Patients can register by providing their name and a unique phone number via a POST request to /patients/register.
/patients/:id/create_report To create a report with a specific status for a patient with a given ID, a POST request should be sent to /patients/:id/create_report.
/patients/:id/all_reports To retrieve all reports of a patient with a given ID, a GET request should be sent to /patients/:id/all_reports. The reports will be listed from oldest to latest.
/reports/:status To list all the reports of all the patients filtered by a specific status, a GET request should be sent to /reports/:status.
